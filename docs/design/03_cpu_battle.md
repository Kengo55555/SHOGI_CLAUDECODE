## 1. 概要

CPU（AI）との対戦機能。3段階のレベルを用意し、初心者から有段者まで幅広く対応する。CPU対戦はサーバーサイドで思考を実行し、結果をクライアントに返す。

---

## 2. AIレベル設計

### 2.1 レベル別仕様

| レベル | 名称 | 探索深さ | 思考時間上限 | 実装方式 |
|-------|------|---------|------------|---------|
| 1 | 初級 | 1〜2手 | 1秒 | Minimax + ランダム性 |
| 2 | 中級 | 3〜5手 | 3秒 | Alpha-Beta + 簡易評価関数 |
| 3 | 上級（MVP簡易版） | 5〜7手 | 5秒 | Alpha-Beta + 改良評価関数 |
| 3+ | 上級（2nd本格版） | - | 10秒 | OSSエンジン（やねうら王）WebAssembly |

### 2.2 初級AI（レベル1）

**設計方針：** 将棋のルールを覚えたての人が勝てるレベル

```
思考アルゴリズム:
1. 全合法手を生成
2. 各手を簡易評価（駒得のみ）
3. 上位50%の手からランダムに選択
4. 10%の確率で完全ランダムな手を指す（弱さの演出）
```

**評価関数（簡易）：**
```typescript
const PIECE_VALUES: Record<PieceType, number> = {
  fu: 100,      // 歩
  kyou: 300,    // 香
  kei: 350,     // 桂
  gin: 500,     // 銀
  kin: 550,     // 金
  kaku: 800,    // 角
  hisha: 1000,  // 飛
  ou: 0,        // 王（計算対象外）
  gyoku: 0,
  tokin: 600,
  narikyou: 550,
  narikei: 550,
  narigin: 550,
  uma: 1050,
  ryuu: 1200,
};

function evaluateSimple(state: BoardState, player: Player): number {
  // 自分の駒の合計 - 相手の駒の合計
  return sumPlayerPieces(state, player) - sumPlayerPieces(state, opponent(player));
}
```

### 2.3 中級AI（レベル2）

**設計方針：** アマチュア級位者程度の棋力

```
思考アルゴリズム:
1. Alpha-Beta探索（深さ3〜5）
2. 反復深化（時間が許す限り深く探索）
3. 簡易評価関数（駒得 + 駒の位置評価 + 王の安全度）
```

**評価関数（中級）：**
```typescript
function evaluateMedium(state: BoardState, player: Player): number {
  let score = 0;

  // (1) 駒得（駒の価値合計の差）
  score += materialScore(state, player);

  // (2) 駒の位置評価（中央寄りの駒に加点）
  score += positionalScore(state, player);

  // (3) 王の安全度（王の周囲の味方駒数）
  score += kingSafetyScore(state, player);

  // (4) 持ち駒の価値（盤上の駒より少し低く評価）
  score += mochigomaScore(state, player) * 0.85;

  return score;
}
```

### 2.4 上級AI（レベル3・MVP簡易版）

**設計方針：** 中級の延長。2ndリリースでOSSエンジンに置き換え予定

```
思考アルゴリズム:
1. Alpha-Beta探索（深さ5〜7）
2. 反復深化 + 時間管理
3. キラーヒューリスティック（良い手を優先探索）
4. クイエス探索（駒の取り合いが続く局面を延長探索）
5. 改良評価関数
```

**追加評価要素：**
- 飛車・角の利き（ライン上のマス数）
- 玉の囲い形状のパターン認識（矢倉・美濃など簡易判定）
- 歩の形（二歩禁止は当然として、歩切れのペナルティ等）

## 4. 実行アーキテクチャ

### 4.1 サーバーサイド実行

```
[クライアント]                    [サーバー]
    │                               │
    ├── POST /api/matches/cpu ─────→│  CPU対戦開始
    │←── { matchId, gameState } ────┤
    │                               │
    ├── move（指し手送信）──────────→│
    │                               ├── 合法性チェック
    │                               ├── 盤面更新
    │←── move_accepted ─────────────┤
    │                               │
    │                               ├── CPU思考開始（非同期）
    │                               │   └── Worker Threadで実行
    │←── cpu_thinking ──────────────┤  （思考中表示用）
    │                               │
    │←── move_accepted（CPU手）─────┤  CPU思考完了
    │                               │
    │   ... 繰り返し ...             │
```

### 4.2 思考時間の演出

- 初級：0.5〜1.5秒（即座に返すと味気ないので最低0.5秒待つ）
- 中級：1〜3秒
- 上級：2〜5秒

思考時間中はクライアントに「CPU思考中...」の表示を出す。

## 5. CPU対戦のゲームフロー

### 5.1 対局開始

```typescript
// POST /api/matches/cpu
interface CpuMatchRequest {
  cpuLevel: 1 | 2 | 3;
  timeControl: 10 | 15;    // 持ち時間（分）
  playerSente: boolean;     // trueならプレイヤーが先手
}
```

**処理：**
1. matchesテーブルにレコード作成（gote_id=NULL, cpu_level設定）
2. 初期GameStateを生成
3. プレイヤーが後手の場合、CPUが先手の初手を即座に思考

### 5.2 待った機能

- CPU対戦のみで使用可能（オンライン対戦では不可）
- 1局あたり最大**3回**まで
- 直前の自分の手とCPUの応手の**2手を戻す**
- 残り回数をUIに表示

```typescript
// POST /api/matches/:id/undo
// レスポンス: { gameState, remainingUndos }
```

### 5.3 盤面リセット

- 対局を途中で中断してやり直す
- matchesテーブルのstatusを`aborted`に更新
- 棋譜は途中まで保存する

## 7. テスト観点

| テスト項目 | 内容 |
|-----------|------|
| 初級AI | 合法手のみを返すこと、思考時間上限内に返すこと |
| 中級AI | 明らかな駒取りを逃さないこと（タダ取りの局面で取る手を選ぶ） |
| 上級AI | 3手詰めの局面で詰ます手を選ぶこと |
| 待った | 3回まで使用可能、4回目はエラー |
| 待った | 正しく2手戻ること |
| 時間管理 | 思考時間が上限を超えないこと |
| 対局開始 | matchesレコードが正しく作成されること |
| 後手選択 | CPUが先手の場合、初手が正しく返ること |
