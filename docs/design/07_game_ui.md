## 1. 概要

将棋の伝統的な美しさを保ちつつ、PC・スマホ両対応の快適な対局UIを実現する。CPU対戦・オンライン対戦で共通の対局画面コンポーネントを使用する。

## 3. 将棋盤（ShogiBoard）

### 3.1 レンダリング仕様

| 項目 | 仕様 |
|------|------|
| 盤面サイズ | PCデフォルト: 450×450px、レスポンシブ可変 |
| 最小マスサイズ | 36×36px（スマホ最小、駒の文字が潰れない閾値） |
| 盤の背景色 | 榧材風の木目色（`#DDB06C` 系、CSSまたはテクスチャ画像） |
| 罫線 | 1px solid `#4A3520`（濃茶） |
| 星（目印） | 4箇所（3-3, 3-7, 7-3, 7-7位置に小さな●） |

### 3.2 座標表示

```
     9   8   7   6   5   4   3   2   1    ← 筋（上部に数字）
   ┌───┬───┬───┬───┬───┬───┬───┬───┬───┐
一 │   │   │   │   │   │   │   │   │   │  ← 段（右側に漢数字）
   ├───┼───┼───┼───┼───┼───┼───┼───┼───┤
二 │   │   │   │   │   │   │   │   │   │
   ...
九 │   │   │   │   │   │   │   │   │   │
   └───┴───┴───┴───┴───┴───┴───┴───┴───┘
```

- 盤面の向きは自分が手前（下側）になるよう自動回転
- 先手が下側、後手が上側（先手視点）
- 後手でログインしている場合は盤面を180度回転

### 3.3 描画方式

**CSS Grid + SVG/画像ハイブリッド：**

```typescript
// ShogiBoard.tsx の構造
<div className="shogi-board-container">
  {/* 盤面背景（木目テクスチャ） */}
  <div className="board-background" />

  {/* 9×9 グリッド */}
  <div className="board-grid" style={{ display: 'grid', gridTemplateColumns: 'repeat(9, 1fr)' }}>
    {squares.map((row, dan) =>
      row.map((piece, suji) => (
        <Square
          key={`${suji}-${dan}`}
          position={{ suji: 9 - suji, dan: dan + 1 }}
          piece={piece}
          isLastMove={isLastMoveSquare(suji, dan)}
          isLegalTarget={isLegalTarget(suji, dan)}
          onClick={handleSquareClick}
          onDragOver={handleDragOver}
          onDrop={handleDrop}
        />
      ))
    )}
  </div>
</div>
```

## 5. 操作方式

### 5.1 駒の移動

**方式1：クリック（タップ）2回方式**
1. 駒をクリック → 合法手のハイライト表示
2. 移動先をクリック → 駒が移動
3. 別の自分の駒をクリック → 選択変更
4. 空きマス or 盤面外クリック → 選択解除

**方式2：ドラッグ＆ドロップ**
1. 駒をドラッグ開始 → 合法手のハイライト表示
2. 移動先にドロップ → 駒が移動
3. 不正な場所にドロップ → 元の位置に戻る

**両方式を同時に有効にする。** スマホではタップ方式が主、PCではドラッグも使える。

### 5.2 持ち駒を打つ

1. 持ち駒エリアの駒をクリック（またはドラッグ）
2. 打てるマスがハイライト表示される
3. 打ちたいマスをクリック（またはドロップ）

### 5.3 成り確認

駒が敵陣に入る（または出る/動く）際に成れる場合：

```
┌──────────────────┐
│  成りますか？      │
│                  │
│ ┌──────┐┌──────┐ │
│ │ 成る  ││不成  │ │
│ │  龍   ││ 飛   │ │
│ └──────┘└──────┘ │
└──────────────────┘
```

- 強制成りの場合はダイアログを表示せず自動的に成る
- ダイアログには成った後の駒と成らない場合の駒を視覚的に表示

### 5.4 操作の状態管理

```typescript
type InteractionState =
  | { type: 'idle' }
  | { type: 'piece_selected'; position: Position; legalMoves: Position[] }
  | { type: 'mochigoma_selected'; piece: KomaType; legalDrops: Position[] }
  | { type: 'dragging'; from: Position | 'mochigoma'; piece: PieceType }
  | { type: 'promotion_dialog'; from: Position; to: Position };
```

## 6. 合法手ハイライト

### 6.1 表示スタイル

| 要素 | スタイル |
|------|---------|
| 移動可能マス（空きマス） | 中央に半透明の●（`rgba(0,0,0,0.2)`、直径40%） |
| 移動可能マス（取れる駒あり） | マス全体に半透明の丸枠（border-radius: 50%） |
| 最終手ハイライト | マス全体に薄い黄色オーバーレイ（`rgba(255,200,0,0.3)`） |
| 王手表示 | 王の駒に赤いグロー効果 |

### 6.2 設定

- 合法手ハイライトのON/OFF切り替え可能（設定画面またはゲーム内トグル）
- デフォルトはON

## 7. 持ち駒表示（MochigomaBar）

### 7.1 レイアウト

```
[相手の持ち駒]
┌─────────────────────────────┐
│ 歩×3  香  桂  銀  金  角  飛  │
└─────────────────────────────┘

         将 棋 盤

┌─────────────────────────────┐
│ 歩×2  銀                     │  ← 自分の持ち駒
└─────────────────────────────┘
```

- 駒画像を縮小して横並び表示
- 同じ種類の駒が複数ある場合は「×N」で表示
- 持ち駒がない場合は「なし」と表示
- 自分の持ち駒はクリック/ドラッグ可能、相手の持ち駒は表示のみ

### 7.2 PC配置

PCではより伝統的な配置として、盤面の右側（自分）と左側（相手）に駒台を配置することも検討：

```
┌───────┐  ┌─────────────────┐  ┌───────┐
│ 相手の │  │                 │  │       │
│ 持ち駒 │  │    将棋盤        │  │       │
│ 歩×3  │  │                 │  │       │
│ 香    │  │                 │  │ 自分の │
│       │  │                 │  │ 持ち駒 │
│       │  │                 │  │ 歩×2  │
│       │  │                 │  │ 銀    │
└───────┘  └─────────────────┘  └───────┘
```

## 8. 残り時間表示（TimeDisplay）

### 8.1 表示仕様

```typescript
interface TimeDisplayProps {
  remainingMs: number;     // 残り時間（ミリ秒）
  isMyTurn: boolean;       // この人の手番か
  timeControl: number;     // 持ち時間（分）
}
```

| 状態 | 表示形式 | スタイル |
|------|---------|---------|
| 残り1分超 | `MM:SS` | 通常 |
| 残り1分以下 | `0:SS` | 赤文字＋点滅 |
| 残り10秒以下 | `0:0S` | 赤太字＋高速点滅＋効果音 |
| 手番中 | 背景色付き | アクティブ感を演出 |
| 相手手番 | 通常表示 | |

## 11. レスポンシブ対応

### 11.1 ブレークポイント

| 画面幅 | 分類 | 盤面サイズ | レイアウト |
|--------|------|-----------|-----------|
| 1024px〜 | PC | 450px | 横並び（盤面+持ち駒+情報） |
| 768〜1023px | タブレット | 380px | 横並び（コンパクト） |
| 〜767px | スマホ | 画面幅-16px | 縦積み |

### 11.2 スマホレイアウト

```
┌──────────────────────┐
│ CPU（中級）  12:34    │
│ [持ち駒: 歩歩]        │
├──────────────────────┤
│                      │
│                      │
│      将棋盤           │
│   （画面幅いっぱい）    │
│                      │
│                      │
├──────────────────────┤
│ [持ち駒: 歩 銀]       │
│ あなた       08:45   │
├──────────────────────┤
│ [待った] [投了] [中断] │
└──────────────────────┘
```

### 11.3 タッチ操作の考慮

- タップ領域は最低44×44px（iOS HIG準拠）
- ドラッグ開始の判定：20px以上の移動で開始（誤操作防止）
- 長押しはなし（誤操作の温床）
- ダブルタップによるズーム防止（meta viewport設定）

## 12. 効果音

| イベント | 効果音 | 説明 |
|---------|--------|------|
| 駒を置く | パチッ | 将棋駒を盤に置く音 |
| 駒を取る | パチッ（やや強め） | 取りの動作 |
| 王手 | パチッ + 短い効果音 | 緊張感の演出 |
| 時間切れ警告 | ティッティッ | 残り10秒の秒読み音 |
| 対局開始 | 短い和楽器音 | |
| 対局終了 | 短い終了音 | |

- 効果音のON/OFF切り替え可能
- 音量調節は不要（ON/OFFのみ）
- 音源はフリー素材またはWeb Audio APIで生成

## 13. テスト観点

| テスト項目 | 内容 |
|-----------|------|
| 盤面描画 | 初期配置が正しく表示されること |
| 駒の移動（クリック） | 選択→移動先クリックで駒が動くこと |
| 駒の移動（ドラッグ） | ドラッグ＆ドロップで駒が動くこと |
| 不正な操作 | 合法手以外の移動が拒否されること |
| 成り確認 | 敵陣での成り確認ダイアログが表示されること |
| 強制成り | 行き所がない場合に自動で成ること |
| 持ち駒操作 | 持ち駒をクリック→打つが正しく動作すること |
| 時間表示 | 残り時間が正確に表示・更新されること |
| レスポンシブ | 各ブレークポイントで正しくレイアウトされること |
| スマホ操作 | タップ操作で正しく駒を動かせること |
